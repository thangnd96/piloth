# Layer Boundary Rule

## Rule

Mọi thay đổi phải nằm đúng responsibility của layer hoặc cross-cutting system.

## Checks

- Policy → Rules & Hooks.
- Context → Memory.
- Fact/documentation → Knowledge.
- Capability/workflow reusable → Skills.
- Orchestration/lifecycle → Runtime.
- Role execution → Agents.
- Integration/adapter → Tools hoặc Adapters.
- Team composition → Agent Teams.
- Approval/risk/escalation → Governance.
- Quality gate/evidence → Evaluation.

Nếu phát hiện sai layer, flag và đề xuất refactor; không âm thầm đặt sai để hoàn thành nhanh.
